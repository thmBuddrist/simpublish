描述

因为简悦基于 Local First 构建的存储体系，其导出文件全部都会存储在本地同步目录的 output 目录下。如果我们开启了导出 HTML 功能，则阅读模式下的文章就会离线备份到本地。

由于这样的机制，我们可以尝试把 output 路径下 HTML 文件托管到网络上，使用成熟的静态文件托管服务把 Local 的文档备份变成在线的资料库，而令人可喜的是，这样的静态托管服务很多，而且大多数都有免费的套餐，个人使用基本不会产生费用。

结合简悦的 自动化 和 外部链接 功能，我们也很容易在阅读文章的过程中，随时把标注的内容输出与输出的外部链接绑定其实，实现从阅读笔记到外部资料库的深度跳转。

教程

准备

  1. Github 账户
  2. Git 环境
  3. Vercel 账户

设置

设置有两种方式：

Deploy 方案

  • 优点：简单
  • 缺点：当主程序更新后，无法自动化更新主程序

Fork 方案

  • 优点：当主程序更新后，可手动自动更新。
  • 缺点：相对复杂
本文以 Deploy 方案为主。

进入 https://github.com/OverflowCat/simpublish 并选择 Deploy

稍等一会会自动进入下面界面的页面

登录你的 Github 账户

稍等片刻后会让你选择 账户名称（一般默认即可），并给这个 Repo 起名，比如我的名字叫做 simpublish-demo，然后选择 Create，稍等片刻后，会进入下图的界面。

设置密码（因为自建资料库一般都是很个人的行为，所以默认是需要密码才能访问的），设置后选择 Deploy，如果之前设置都正确的话，会看到下面的界面

稍等一会会进入下面的界面

如果之前设置都正确的话，会看到下面的界面

点击 Go to Dashboard 进入下图页面后选择 Visit

进入 https://<你之前设置的名称>.vercel.app 比如我的就是 https://simpublish-demo.vercel.app 输入之前设定的密码

最终会看到下图的界面

发布

进入 https://vercel.com/<你的 Vercel 账户>/<起的名字>/settings/git ，打开在 Github 上面的 repo

点击 Code 复制 repo 地址

在本地使用 git clone git@github.com:<username>/<title>.git Clone 到本地。

手动建立 _output （如果没有的话）

至此，全部的环境已经准备完毕，当 _output 出现 HTML 文件后，使用 git 提交后，进入 https://simpublish-demo.vercel.app 就能看到你刚才发布的内容了。

导出标注

在任意阅读页面进行标注操作，然后自动导入到 Obsidian 并将外部链接指向刚刚设置的资料库。

简悦中有一个功能叫做【外部链接】，为了让简悦的【外部链接】指向刚刚建立的资料库，需要做一些设置：

  1. 首先需要安装 / 升级 Markdown 辅助增加插件 到最新版。
  2. 然后配置【简悦】→【服务】→【外部链接】为 https://<site_name>.vercel.app/articles/{{id}} （ 根据上面的例子，这个链接应该是 https://simpublish-demo.vercel.app/articles/2073
  3. 在【Markdown 辅助增加插件】的模板中可以使用 {{an_ext_uri}} 占位符作为外部链接。

将上面的内容复制到选项页 → 服务 → 定制导出

然后在 Markdown 辅助增强插件中就会实现下图的功能。
